# 刷题逻辑优化方案

## 阶段1：基础进度记录 ✅ 已完成

### 实现内容
1. **Store 新增状态**：
   - `examProgress: Record<string, number>` - 记录每个试卷的题组索引
   - `practiceModeProgress: Record<PracticeMode, number>` - 记录每个模式的题组索引
   - `saveExamProgress()` - 保存试卷进度
   - `savePracticeModeProgress()` - 保存模式进度

2. **Practice.tsx 修改**：
   - 根据 `initialQuestionId` 判断是否是真题套卷模式
   - 真题套卷模式只显示该套卷的题目
   - 提交答案时保存进度到下一题（但不跳转页面）
   - 点击"下一题"时也保存进度
   - 初始化时从保存的进度恢复

3. **ExamSets 组件修改**：
   - 构建题组数组（与 Practice.tsx 逻辑一致）
   - 从 `examProgress` 恢复进度，获取正确的题目
   - "继续练习"按钮从保存的进度开始

### 用户体验
- 做完题目提交答案后，可以查看解析
- 切换到其他页面再回来，会从下一题开始
- 真题套卷和刷题模式的进度互不干扰
- 每个刷题模式有独立的进度记录

## 阶段2：智能刷题模式 ✅ 已完成

### 实现内容

#### 1. 刷题模式选择
在 Practice.tsx 顶部添加模式切换按钮：
- **顺序刷题** - 按题目顺序做题
- **只刷未做题** - 只显示从未做过的题目
- **只刷错题** - 只显示到达复习时间的错题（基于艾宾浩斯遗忘曲线）
- **智能刷题** - 综合排序：复习时间错题 > 未做 > 知识点正确率低

#### 2. 艾宾浩斯遗忘曲线
复习间隔：1天、2天、4天、7天、15天
- **只刷错题**：只显示到达复习时间的错题
- **智能刷题**：优先推荐到达复习时间的错题，做对的题不出现

#### 3. 知识点正确率分析
智能刷题会计算每个知识点的正确率，优先推荐正确率低的知识点题目

## 阶段4：学习路径优化 ✅ 已完成

### 实现内容

#### 1. 薄弱知识点识别
创建 `WeakPoints.tsx` 组件：
- 分析 answerHistory，计算每个知识点的正确率
- 筛选做题数≥3且正确率<60%的知识点
- 按正确率从低到高排序，显示 Top 10
- 提供"针对练习"按钮，直接跳转到该知识点题目

#### 2. 每日推荐复习
创建 `DailyRecommendations.tsx` 组件：
- 基于艾宾浩斯遗忘曲线（1、2、4、7、15天）推荐复习题目
- 错题优先：到达复习时间的错题优先级最高
- 巩固复习：3天前和7天前做对的题目也会推荐复习
- 显示推荐理由和天数
- 提供"开始复习"按钮，一键开始复习

#### 3. 集成到统计页面
在 Statistics.tsx 中集成两个新组件：
- 放置在总览卡片之后
- 优先显示今日推荐复习
- 其次显示薄弱知识点识别

### 用户体验
- 智能识别学习薄弱环节
- 科学安排复习计划
- 一键开始针对性练习
- 提高学习效率

## 技术实现要点

### Store 扩展
```typescript
interface AppState {
  // 刷题模式
  practiceMode: 'sequential' | 'undone' | 'wrong' | 'smart'
  setPracticeMode: (mode: PracticeMode) => void

  // 今日统计
  todayStats: {
    date: string
    count: number
    correct: number
    timeSpent: number
  }
  updateTodayStats: (correct: boolean, timeSpent: number) => void
}
```

### 性能优化
- 使用 useMemo 缓存筛选结果
- 题目导航器使用虚拟滚动（react-window）
- 大数据量时分页加载

### UI/UX 优化
- 模式切换使用 Tab 组件
- 题目状态使用图标+颜色双重标识
- 添加过渡动画提升体验
